ITIS 3135 Client Project Summary

Meeting the Specifications

There are 3 diffrent ways I use interactive Java Script in my client project. The first way is my dynamic headers and footers. I only have the header and footer of this website writen in one place, in a components file. Java Script loads these files onto each of the pages of this website. I also use Java Script for the gallery. In this gallery, I, by default, display many lower quality versions of photos taken by my client. Once one of these photos is clicked, the gallery/slideshow pops up. This slideshow uses higher quality photos showing the detail of the photos. I use Java Script to change the src of the photo being displayed depending on the input you choose. My third way of using Java Script in my client project is input encoding. When a user puts an input into the forms that contains characters that could manipulate the code, it is encoded. This use of Java Script makes it so that the code would not execute. Once the input is encoded, an alert pops up telling the user that their input was encoded.

I followed the CRAP principles by:

  1. C:
    • Picking colors that contrast well that eventually passed the validation (it took me many iterations of diffrent colors to pass this validation).
  2. R:
    • Put elements that repeat over pages in the same spots (i.e, the header and footer).
  3. A:
    • Aligned all things in a proper and set way.
  4. P:
    • Made relating items close to each other.
